🦅 Shutter Heroes

생물 목격 정보 공유 플랫폼 - AWS EC2 Single Server Architecture

Nginx: Reverse Proxy (80, 443)
Frontend: React + Vite (3000)
Backend: Spring Boot (8080)
YOLO: FastAPI AI (8000)
PostgreSQL: Database (5432)
AWS S3: Image Storage

🏗️ 아키텍처 특징

  • Single Server Deployment: AWS EC2 하나의 인스턴스에서 모든 서비스를 Docker Compose로 관리
  • Container Orchestration: 6개 컨테이너 (Nginx, Frontend, Backend, YOLO, PostgreSQL, Certbot + Migrator)
  • Domain Management: 가비아에서 구매한 도메인 (shutter-heroes.site, api.shutter-heroes.site)
  • SSL/TLS: Let's Encrypt를 통한 무료 SSL 인증서 자동 갱신 (Certbot)
  • Image Processing: S3 업로드 → OpenAI Vision API + YOLO AI로 이중 동물 인식
  • Spatial Data: PostGIS 확장을 통한 GPS 좌표 기반 공간 쿼리 지원
  • Authentication: Kakao OAuth2 소셜 로그인 + JWT 토큰 기반 인증
  • ECR Integration: AWS ECR에서 컨테이너 이미지 pull (Frontend, Backend, YOLO)
  • Network Isolation: 모든 컨테이너가 app-network (bridge) 내에서 내부 통신
  • Health Checks: 각 서비스별 헬스체크로 자동 재시작 관리

🔐 보안 구성

  • HTTPS Only: Nginx가 HTTP (80) → HTTPS (443) 리다이렉트
  • Internal Ports: 모든 서비스는 내부 포트만 노출, 외부는 Nginx를 통해서만 접근
  • EXIF Removal: S3에 업로드 시 원본과 EXIF 제거 버전 2개 저장 (개인정보 보호)
  • SSH Access: MobaXterm을 통한 안전한 서버 관리 (포트 22)
  • Environment Variables: 모든 민감 정보는 .env 파일로 관리 (Git에서 제외)

📊 사용 기술 스택

  • Infra: AWS EC2, S3, ECR / Gabia Domain / Let's Encrypt SSL
  • Frontend: React, Vite, TypeScript
  • Backend: Spring Boot 3.5.6, Java 21, Spring Security, JWT, OAuth2
  • Database: PostgreSQL 15 + PostGIS 3.4 (공간 데이터)
  • AI: OpenAI Vision API (GPT-4), YOLOv8 (FastAPI), Python
  • DevOps: Docker Compose, Nginx, Certbot, MobaXterm (SSH)
  • External APIs: Kakao OAuth2, OpenAI GPT-4
